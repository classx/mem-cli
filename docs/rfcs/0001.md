## Problem

В каждой новой сессии требуется повторно собирать контекст проекта вручную.
Это замедляет работу и повышает риск расхождения решений между сессиями.

## Goal

Создать минимальный проект на Rust + SQLite, который будет хранить и быстро
восстанавливать долговременный контекст проекта между сессиями.

## Design

1. Базовая структура Rust-приложения (CLI).
2. SQLite-файл `project_context.db` в корне проекта (или в `data/`).
3. Минимальная схема:
   - `facts` (стабильные факты о проекте),
   - `decisions` (принятые технические решения),
   - `commands` (проверенные команды build/test/lint),
   - `modules` (краткое описание модулей и их ответственности).
4. Команды CLI:
   - базовый формат: `mem-cli [init|add|list|decisions|modules] [facts|decisions|commands|modules]`.
   - `init` — создать БД и схему (без обязательного второго аргумента).
   - `add` / `list` — универсальные действия для сущностей `facts|decisions|commands|modules`.
   - `decisions` и `modules` — специализированные режимы/алиасы для быстрого доступа к этим сущностям.
   - для операций удаления используется единый паттерн soft/hard delete:
     `delete` = установка `deleted_at`, `purge` = физическое удаление.
   - основной интерфейс работы с данными — через `mem-cli`;
     прямые SQL-запросы к SQLite допустимы только для отладки и диагностики.
5. Версионирование схемы через таблицу `schema_version`.

## Alternatives

1. Использовать только текстовые файлы (`AGENTS.md`, README, заметки).
   Плюс: просто. Минус: хуже структурированный поиск и слабая масштабируемость.
2. Использовать только встроенную memory Copilot.
   Плюс: быстро для персональных предпочтений. Минус: не заменяет
   проектную структурированную базу знаний.
3. Использовать внешнюю СУБД (PostgreSQL).
   Плюс: масштаб. Минус: избыточно для локального/небольшого проекта.
