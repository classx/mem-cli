# PLAN-5: Реализация RFC-0005 (MCP-сервер для mem-cli)

Реализация [RFC-0005](rfcs/0005.md). План разбит на фазы; каждая фаза — отдельный
branch, каждая задача — sub-branch `phase-5/X.Y-description`.

Статус: **выполнено** (v1 MCP tools + resources, документация и релизные правки).

## Фаза 1 — Подготовка и каркас

- **1.1** Уточнить минимальный scope v1 (набор MCP tools из RFC-0005).
- **1.2** Добавить каркас MCP-модуля в `src/` без изменения существующего CLI UX.
- **1.3** Добавить подкоманду `mem-cli mcp` в `src/main.rs` (запуск по `stdio`).
- **1.4** Зафиксировать ограничения v1: без произвольного SQL/shell, только
  операции, эквивалентные текущим командам.

Критерий завершения фазы: `mem-cli mcp` стартует и корректно инициализирует сервер.

## Фаза 2 — Общий core-слой для CLI и MCP

- **2.1** Выделить/дочистить общие обработчики доменных операций (без дублирования).
- **2.2** Перевести CLI-вызовы на общий слой там, где это необходимо.
- **2.3** Подключить MCP-обработчики к тому же слою (`src/db.rs` + общая валидация).

Критерий завершения фазы: одинаковая логика валидации и ошибок для CLI и MCP.

## Фаза 3 — MCP tools v1

- **3.1** Реализовать read-tools:
  - `list_facts`, `list_decisions`, `list_modules`, `list_commands`
  - `find_by_tag`
- **3.2** Реализовать write-tools:
  - `add_fact`, `add_decision`, `add_module`, `add_command`
  - `tag_record`, `untag_record`
- **3.3** Реализовать `doctor` как MCP tool.
- **3.4** Описать входные и выходные JSON-схемы для каждого tool.

Критерий завершения фазы: все tools из v1 доступны и возвращают стабильный JSON.

## Фаза 4 — Ошибки, логирование, стабильность интерфейса

- **4.1** Унифицировать формат ошибок MCP (валидация, not found, internal).
- **4.2** Проверить, что протокольный вывод остаётся в `stdout`, логи — в `stderr`.
- **4.3** Зафиксировать правила совместимости MCP tool-схем для minor-релизов.

Критерий завершения фазы: предсказуемые ошибки и чистый протокол без шумовых логов.

## Фаза 5 — Тестирование

- **5.1** Обновить/добавить unit-тесты общего core-слоя.
- **5.2** Добавить интеграционные тесты старта `mem-cli mcp` и базовых tool-вызовов.
- **5.3** Добавить smoke-сценарии на read/write операции и обработку ошибок.
- **5.4** Прогнать проверки через Makefile:
  - `make build`
  - `make test`
  - `make lint`

Критерий завершения фазы: зелёные build/test/lint и воспроизводимые интеграционные тесты MCP.

## Фаза 6 — Документация и релиз

- **6.1** Обновить `README.md`: запуск `mem-cli mcp`, примеры подключения MCP-клиентов.
- **6.2** Обновить `AGENTS.md`: рекомендации по использованию MCP-интерфейса.
- **6.3** Обновить `CHANGELOG.md` (English) и версию в `Cargo.toml` (minor bump).
- **6.4** Подготовить merge в main по git-workflow проекта.

Критерий завершения фазы: документация синхронизирована с реализацией, релиз готов.

## Риски и контроль

- **Риск:** дублирование логики между CLI и MCP.
  **Контроль:** единый core-слой и тесты на идентичное поведение.
- **Риск:** нестабильный формат tool-ответов.
  **Контроль:** фиксированные JSON-схемы и интеграционные проверки.
- **Риск:** протокольный шум в stdout.
  **Контроль:** строгий раздел stdout/stderr и smoke-тест старта.
